decode()method 
public function decode($oggBytes:ByteArray, $samplesPerLoop:int = 2048, $delayPerLoopInMS:int = 33, $progressNotifications:Boolean = true):void

Kicks off a decode of the $oggBytes Chunks of this are executed on a timer.

Parameters

$oggBytes:ByteArray — - ogg vorbis formated bytes to decode
 
$samplesPerLoop:int (default = 2048) — - number of samples to decode per timer tick
 
$delayPerLoopInMS:int (default = 33) — - number of milliseconds to wait before kicking off another decode loop
 
$progressNotifications:Boolean (default = true) — - if true, the decoder notifies flash of progress, and dispatches progress events. With this enabled, it might slow down the decode a touch.

decodeOggSample()method 
public function decodeOggSample($dataPtr:Number, $numSamplesPerLoop:int, $blockAlign:Number, $sampleMultiplier:Number):Object

Decode exactly one set of samples. This is the finest granularity offered by this wrapper for decoding. This can be used on its own to decode small chunks. This would be useful for decoding large files on the fly so you don't have to wait for the entire file to be decoded. This would work well inside of a Sound SAMPLE_DATA event handler. Just be sure to call initDecoder() before using this on its own.

Parameters

$dataPtr:Number — memory address of the audio data in Alchemy
 
$numSamplesPerLoop:int — number of samples to process before Alchemy yields back to flash
 
$blockAlign:Number — Set from AudioInfo
 
$sampleMultiplier:Number — Set from AudioInfo

Returns
Object — returns object from Alchemy that contains position, length, and total_bytes_read. This data is used for keeping track of progress
encode()method 
public function encode($rawBytes:ByteArray, $oggComments:OggComments = null, $oggQuality:Number = 0.5, $wavLoopsPerYield:int = 10, $vorbisLoopsPerYield:int = 1, $progressNotifications:Boolean = true):void

This method actually does the encoding. Before calling this method, it is suggested that you set up listeners for OggManagerEvent.ENCODE_PROGRESS as well as OggManagerEvent.ENCODE_COMPLETE The first thing this does, is do a conversion to WAV. When that is finished, the actuall ogg vorbis encoding starts.

Parameters

$rawBytes:ByteArray — 44.1kHz, 16bit, 2 Channel wav formatted bytes
 
$oggComments:OggComments (default = null) — Value Object that holds the comment tags we want saved with the Ogg file.
 
$oggQuality:Number (default = 0.5) — Value can be from -0.1 to 1.0. Bigger numbers bigger files better quality.
 
$wavLoopsPerYield:int (default = 10) — Number of wav chunks processed before control is given back to Flash. Higher numbers slow down flash, but speed up encoding.
 
$vorbisLoopsPerYield:int (default = 1) — Number of pages processed before control is given back to Flash. Higher numbers slow down flash, but speed up encoding.
 
$progressNotifications:Boolean (default = true) — set true to get progress events during the encode. This may slow down the encode slightly.

Sets up the decoder and grabs memory pointer from alchemy that points to where the audio data is stored in memory. This must be called before starting a decode process. This is handled automatically unless you are using decodeOggSample() manually.
